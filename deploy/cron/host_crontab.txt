# Canonical host crontab for the Mancini bot VM (152.70.113.24).
# Apply with:  crontab /home/ubuntu/mancini/deploy/cron/host_crontab.txt
#
# ALL TIMES BELOW ARE UTC. The VM host runs in UTC and Ubuntu's cron
# daemon schedules in system time — a `TZ=America/New_York` line in the
# crontab does NOT change when jobs fire (it only sets the jobs'
# environment). The 2026-06-03 "fix" that added that line never worked:
# every job kept firing 4-5 hours early in ET terms, so the 17:30 "ET"
# extraction ran at 13:30 ET, fetched yesterday's post, and published it
# mislabeled as tomorrow's plan. Discovered 2026-06-11.
#
# Defense in depth: live/mancini_llm_extract.py now validates the post's
# plan date against the target trading date and skips stale posts, so an
# early run is harmless — it logs "stale_post" and a later run retries.
#
# ET equivalents drift with DST (EDT = UTC-4, EST = UTC-5).

# Nightly retrospective + report — 22:05/22:10 UTC = 18:05/18:10 EDT
